
### CRM API Gateway ‚Äî Key API Decisions

Single public entry point (`apps/crm/api-gateway`) for the React SPA and future EHR integration ‚Äî pure Fastify reverse proxy, no DB schema, no EventBridge subscriptions. **Not proxied:** Identity Service (browser calls directly), platform services (`@platform/*` UI components call directly), Media Service (browser uses presigned PUT). **Route prefixes:** `/v1/leads`, `/v1/pipeline`, `/v1/conversations`, `/v1/campaigns`, `/v1/referrals`, `/v1/reports`, `/v1/imports`, `/v1/notifications`. **Versioning:** `/v1/` prefix stripped before forwarding; full path after `/v1` preserved exactly. **Per-service Fastify plugin** structure (one file per downstream service). **Three auth paths:** (1) Public ‚Äî `GET /health` + 3 Referral routes (no JWT, IP rate-limit only); (2) JWT ‚Äî Bearer non-`ak_`-prefixed, `@ortho/auth-middleware` verifies via JWKS at `{IDENTITY_SERVICE_URL}/.well-known/jwks.json`, JWKS re-fetched on unknown `kid` (max 3 retries, back-off capped 5s, returns 401 while unreachable), JWT forwarded unchanged downstream; (3) API key ‚Äî `ak_`-prefixed Bearer detected, validated via `POST /identity/api-keys/validate` (VPC-only, `X-Internal-Secret`), response `{ permissions: [] }` cached by `SHA256(raw_key)` for 60s, `last_used_at` updated on cache miss only, Identity Service outage ‚Üí fail closed `503 auth_unavailable`. **Downstream forwarding for API key requests:** `Authorization` header stripped; `X-Api-Key-Permissions: <comma-separated>` injected instead ‚Äî downstream services don't validate `ak_` keys independently. **Exception:** gateway's own VPC call to Lead Service for channel resolution sends raw `ak_` key directly (service-to-service pattern, Lead Service validates it). **Header security:** incoming `X-User-*` and `X-Api-Key-Permissions` headers stripped before injection; `X-Request-ID` always generated fresh (client value discarded). **Injected headers (JWT routes):** `X-User-Id` (sub), `X-User-Role` (role), `X-User-Locations` (comma-separated ‚Äî **omitted entirely** when `locations[]` is empty, i.e., marketing/super_admin roles). **Rate limiting:** IP 60 req/min (public, using ALB-injected rightmost IP), `sub` 300 req/min (JWT), `keyHash` 600 req/min (API key). **Pipeline RBAC:** `override: true` on `POST /v1/pipeline/transitions` blocked for `call_center_agent` + `marketing_staff` (returns 403); `override: false` / absent treated identically (no check). **Channel resolution on `/convert`:** `lead_id` from request body ‚Üí `GET /leads/:lead_id` (via `LEAD_SERVICE_API_KEY`) ‚Üí inject `channel` into forwarded body (always overwrites client-supplied value); failures: `404` ‚Üí `lead_not_found`, `5xx/timeout` ‚Üí `502`, `null/invalid/malformed channel` ‚Üí `422 channel_resolution_failed`. **SSE proxy** (`GET /v1/notifications/stream`): no timeout, `Connection: keep-alive`, `X-Accel-Buffering: no`, `Last-Event-ID` passed through. **Referral redirect** (`GET /v1/referrals/r/:code`): `followRedirects: false` ‚Äî gateway must return the 302 to the browser, not follow it. **Body limits:** 1MB default, 5MB for `/v1/imports/upload`. **`triggered_by`:** JWT callers use `X-User-Id` forwarded header; API key callers (Automation Engine, Data Import) include `triggered_by` in request body directly.
