
### Data Import Service — Key API Decisions

Product-layer Ortho2 CSV bridge (`apps/crm/import`, schema `crm_imports`). Temporary until EHR integration. **Import types:** `active_patients` (→ contract_signed + convert to In Treatment), `completed_patients` (→ treatment_complete + convert to In Retention), `scheduled_appointments` (→ exam_scheduled + create appointment), `no_shows` (→ contacted, reason: no_show). **Async BullMQ** — single `import-job` type with `phase: parse_match | execute | undo`; `attempts: 1` (no blind retry); frontend polls `GET /imports/:id`. **Membership lookup** — all execution steps call `GET /pipeline/memberships?lead_id={id}&pipeline={expected}&status=active` on Pipeline Engine (not Lead Service — membership ID not on lead object); no active membership → row `failed: "no_active_membership"`. **Stage guards** — skip transition and go directly to `/convert` if lead is already at `contract_signed` (active_patients) or `treatment_complete` (completed_patients); skip transition if already at `exam_scheduled` (scheduled_appointments, avoids duplicate event). **override: true always** — pass uploading coordinator's user UUID as `triggered_by`; imports are human-initiated. **No EventBridge events published** — downstream effects via Pipeline Engine's `lead.stage_changed` + `lead.converted`. **Full revert undo** — `before_snapshot` stored per row before executing; 2-hour window; conversion undo = `POST /pipeline/memberships/:id/close` (sets `status='closed'`, `closed_reason='import_undo'`) + re-enroll via `POST /pipeline/memberships` with `location_id: import.location_id`; transition undo = Pipeline Engine transition with `reason: "import_undo"`; undo uses atomic CAS + follow-up SELECT to discriminate 422/409/404. **Global column mapping** — one saved mapping per import type; `confirm` does NOT re-run match logic. **Batch prefetch** — builds `Map<phone, lead[]>` + `Map<email, lead[]>` from bulk `GET /leads?phones[]=...` + `GET /leads?emails[]=...`; Tier 3/4 name search cached per-row only. **Direct service calls** via `IMPORT_SERVICE_API_KEY`. **Route registration** — `GET /imports/column-mappings/:type` registered before `GET /imports/:id`. **Pending amendments:** Pipeline Engine spec (add `POST /pipeline/memberships/:id/close`; add `"import_undo"` to both `pipeline_stage_history.reason` + `pipeline_memberships.closed_reason` enums); Lead Service spec (`date_of_birth` field, `DELETE /appointments/:id`, bulk `phones[]`/`emails[]` params); arch doc (`IMPORT_SERVICE_API_KEY`, REST call table); Automation Engine spec (`reason: "no_show"` in trigger catalog).
