## Codebase Patterns
- Arize Phoenix npm package is `@arizeai/openinference-instrumentation-anthropic` (no dash in arizeai), NOT `@arize-ai/...`
- All platform services use ESM (`"type": "module"`) with NodeNext module resolution
- Logger: `import { createLogger } from '@ortho/logger'` — Pino-based, Datadog-compatible
- Dockerfile pattern: node:24-alpine, COPY package*.json, npm ci --omit=dev, COPY dist/, CMD node dist/index.js
- knexfile.ts uses `process.env['DATABASE_URL']` (bracket notation for strict TS)
- Service package names: `@platform/<service-name>`
- Run commands from inside the service directory (e.g. `apps/platform/ai/`)
- Migration pattern: `CREATE SCHEMA IF NOT EXISTS`, then `knex.schema.withSchema()` for tables, `knex.raw()` for indexes with fully qualified names
- pg Pool search_path: `new Pool({ connectionString, options: '-c search_path=platform_ai' })` — allows bare table names in queries
- Repository pattern: `createXxxRepository(pool: Pool)` returns object with methods — use `createPool()` helper from completions.ts for pool creation
- Cache pattern: `computeCacheKey()` uses SHA256 of sorted-key JSON; `createCompletionCache(repo)` wraps L1 (lru-cache) + L2 (Postgres repo)
- Anthropic SDK error classes: import from `@anthropic-ai/sdk/error` — `RateLimitError` (429), `InternalServerError` (5xx), `APIError` (base). No `APIStatusError` exists.
- OpenTelemetry: `trace.getTracer('ai-service')` from `@opentelemetry/api` for custom spans
- Fastify decorators need `src/types.d.ts` with `declare module 'fastify' { interface FastifyInstance { ... } }` for type-safe access
- env.ts pattern: `@sinclair/typebox` Value.Check + Value.Errors, PORT parsed with parseInt, defaults to 3000
- Route plugins are async functions `(app: FastifyInstance) => Promise<void>` registered via `app.register()`
- app.ts pattern: `buildApp(pool: Pool): Promise<FastifyInstance>` — registers sensible, decorates deps, registers routes
