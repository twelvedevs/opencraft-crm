## Codebase Patterns
- Route plugins accept `opts: { db: Knex; eventBus: EventBus }` — always destructure and pass both
- When adding new repository calls to a service, update the vi.mock() block in existing unit tests to include the new exports
- Integration tests use `EventBusImpl(new MockDriver())` from `@ortho/event-bus` (factory doesn't support 'memory' driver)
- `buildTestApp()` in `test/integration/helpers.ts` constructs its own Fastify app (doesn't use `buildApp` from src)
- All route signatures updated to `{ db, eventBus }` opts even if eventBus is unused in that route currently
- Publisher functions follow pattern: `bus.publish({ event_type, entity_type: 'lead', entity_id, payload })`
- Service functions that mutate state accept `eventBus: EventBus` param — routes destructure from opts and pass it through
- Activity inserts wrapped in try/catch + console.warn to avoid blocking the main operation on timeline failures
- Merge service uses `MergeError` class with `statusCode` property for tagged errors (404/400/403/503) — routes check `instanceof MergeError` and use `err.statusCode`
- Use `knex.transaction(async (trx) => {...})` for multi-step DB operations; pass `trx` instead of `db` to repository functions inside transactions
- Pipeline Engine URL is in `env.PIPELINE_ENGINE_URL`; auth via `Bearer ${env.SERVICE_AUTH_TOKEN}`
- `AbortSignal.timeout(5000)` for external service calls with 5s timeout
- Mocking Knex transactions: `trx` must be a callable `vi.fn()` (for table access) with `.raw()` and `.fn` properties attached
- Use `vi.stubGlobal('fetch', vi.fn())` to mock native fetch in unit tests; `vi.mock('../../src/env.js')` to mock env vars
- Phone numbers in tests must be valid US numbers — use real area codes (212, 310, etc.), not 555 prefix
