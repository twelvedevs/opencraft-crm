## Codebase Patterns
- Action workers follow createXxxProcessor(repo, queue, fetchFn) factory pattern
- fetchFn defaults to globalThis.fetch — injected in tests for mocking
- Active hours check in send_message/send_email happens BEFORE marking 'running'; in call_ai it happens AFTER the AI call (intentional: AI call always executes immediately)
- Step chain pattern: if nextStepId → findStepById → if non-null → queue.add; else → updateExecutionStatus('completed')
- updateStepStatus 'completed' accepts extras.output for storing JSON result
- Test mock job: moveToDelayed must be added as vi.fn().mockResolvedValue(undefined) to the job object
- vi.mock for active-hours must use factory returning { computeDelay: vi.fn() }; import mocked fn after vi.mock call
- tsconfig only includes src/**/* — test files are not typechecked by tsc
- resolveParams treats any string matching /^[a-zA-Z_][\w.]*$/ as a dot-path reference into eventCtx — use strings with hyphens/numbers in test data to avoid unintentional resolution (e.g. 'static-header-value' not 'value')
