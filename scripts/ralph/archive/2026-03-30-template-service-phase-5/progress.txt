## Codebase Patterns
- Template service uses ESM (`"type": "module"`); import migration `.js` extensions resolve to `.ts` source files in TypeScript.
- Auth plugin: `isServiceApiKey` checks `token.startsWith('ak_')` ‚Äî any `ak_*` string works as service API key.
- JWT signing uses `new TextEncoder().encode(secret)` as key (not `createSecretKey`) ‚Äî must match in tests.
- Integration test helpers: use `port: 0` to get OS-assigned port, derive `serverUrl` from `app.server.address()`.
- Manager tokens need BOTH roles: `['marketing_staff', 'marketing_manager']` ‚Äî the global `requireRole('marketing_staff')` preHandler in templateRoutes blocks manager-only tokens.
- `created_by` column is UUID type ‚Äî JWT `sub` must be a valid UUID (e.g. `'00000000-0000-0000-0000-000000000001'`). Service API keys bypass auth but are blocked by `requireRole`.
- Render route uses `requireAuth` (not `requireRole`) ‚Äî service API keys work there; all CRUD routes use `requireRole('marketing_staff')`.
- Multiple integration test files sharing a DB require `fileParallelism: false` in vitest.config.ts to avoid schema reset race conditions.
