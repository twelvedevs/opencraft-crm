## Codebase Patterns
- CRM API Gateway has no DB, no EventBridge, no BullMQ — purely Fastify + @fastify/reply-from + @fastify/rate-limit
- Service name in package.json uses `@ortho/crm-api-gateway` (not `@crm/` prefix)
- All services: `"type": "module"`, `"module": "NodeNext"`, `"moduleResolution": "NodeNext"`, target ES2022
- Dockerfile uses multi-stage build: builder (full npm ci + tsc) → runtime (npm ci --omit=dev + dist/)
- @ortho/auth-middleware exports `authPlugin` with JWKS-based JWT verification (not used directly in gateway — gateway has its own auth plugin)
- CRITICAL: Route plugins (leads, pipeline, referrals, etc.) must NOT be wrapped with fp(). Using fp() on a route plugin causes routes to register at the ROOT scope without the prefix, making wildcard /* catch all requests instead of specific named routes. Only infrastructure plugins (auth, rate-limit, error-handler, request-id) should use fp().
