## Codebase Patterns
- @ortho/* packages use ESM (`"type": "module"`), NodeNext module resolution, ES2022 target
- Import paths in src/ use `.js` extensions for ESM compatibility
- Migration files follow `NNN_<description>.ts` naming, export `up(knex)` and `down(knex)`
- PostgreSQL COUNT(*) returns bigint as string in node-postgres — use parseInt() or CAST AS int
- Typecheck is run per-service: `npm run typecheck` from the service directory
- Use `import { Redis } from 'ioredis'` (named export) — default import has no construct signatures with NodeNext
- Guard `main()` with `process.argv[1] === fileURLToPath(import.meta.url)` to prevent auto-execution when vitest imports src/index.ts
