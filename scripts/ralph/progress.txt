## Codebase Patterns
- @ortho/logger package: thin pino wrapper; use `createLogger('service-name:component')` — returns a Logger instance with Datadog-compatible `status` field
- Logger import: `import { createLogger } from '@ortho/logger';` then `const log = createLogger('email-service:component');` at module scope
- Log levels: info for request entry/completion, debug for cache hits/misses, warn for business-rule drops, error for all catch blocks (pass full `err` object as first binding)
- Pino log format: `log.info({ context_field: value }, 'message')` — first arg is bindings object, second is message string
