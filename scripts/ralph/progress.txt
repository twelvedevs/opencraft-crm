## Codebase Patterns
- Use `required()`/`optional()` helper pattern in env.ts for env var parsing
- `buildApp()` is async, returns FastifyInstance; registers sensible + authPlugin
- db.ts exports default knex instance + `destroy()` function
- index.ts uses top-level await for `buildApp()` and `app.listen()`
- SIGTERM handler: `app.close()` → `destroy()` → `process.exit(0)`
- vitest.config.ts uses `pool: 'forks'` with `singleFork: true`
- Schema searchPath pattern: `['<schema_name>', 'public']`
- knexfile.ts: `migrations.directory = './migrations'`, `schemaName` for tracking table, `loadExtensions: ['.ts']`
- Migrations: use `withSchema()` for table creation, `knex.raw()` for CHECK constraints and advanced indexes
- GIN trigram: expression indexes need double parens `USING gin ((expr) gin_trgm_ops)`
- Auth middleware: `authPlugin({ jwksUrl, allowedPaths })` — allowedPaths skip JWT validation
- Logger: `createLogger('<service-name>')` returns pino instance, cast to `FastifyBaseLogger`
- Repository functions: standalone exports (not class), `db: Knex` as first arg, use fully qualified table names (`crm_leads.leads`)
- Cursor pagination: base64 JSON `{ lastSeenId, lastSeenSortValue }`, fetch `limit+1` rows, pop last to detect next page
- Qualify columns with table prefix when JOINs are involved (e.g., `crm_leads.leads.id`)
- Service functions: standalone exports, `db: Knex` first arg, wrap repository calls with business logic
- parsePhoneNumber from libphonenumber-js: use 'US' default region, check `.isValid()`, format with `.format('E.164')`
- `npm run test` exits code 1 when no test files found — typecheck is quality gate until test stories
- Route plugin pattern: `async function(app, opts: { db: Knex })`, register in app.ts with `await app.register(fn, { db })`
- `requireRole(['role1', 'role2'])` from @ortho/auth-middleware for preHandler role checks; manager+ = `['call_center_manager', 'marketing_staff', 'marketing_manager', 'super_admin']`
- Import bare `'@ortho/auth-middleware'` in route files to get `req.user` type augmentation
- Register literal path routes BEFORE parameterized routes in Fastify (e.g., `/leads/duplicates` before `/leads/:id`)
