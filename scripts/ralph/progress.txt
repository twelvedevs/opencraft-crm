## Codebase Patterns
- Knex raw for COALESCE: `this.db.raw('COALESCE(col, ?)', [value])` — used for webhook forward-only timestamp updates
- Forward-only guard pattern: use `.whereIn('status', [...allowed])` or `.whereNotIn('status', [...blocked])` before `.update()` — no separate read needed
- New click repositories follow same constructor pattern: `constructor(private readonly db: Knex) {}` + randomUUID() for id
- spamreport on transactional EmailSend → status becomes 'bounced' (not 'spam_reported') per spec Q7 design decision
- AWS SDK mocking in vitest: define `mockSend = vi.fn()` outside `vi.mock()` factory, reference it from within, expose as `__mockSend`; or just define outside and close over it (simpler: put mock outside, use `vi.mock` factory that refs it)
- `vi.resetModules()` does NOT reset mock call counts — use `mockSend.mockClear()` in `beforeEach` instead
- vitest.config.ts `env` block must be updated when env var names change in env.ts (route tests import env.ts at module level)
- Raw body capture for ECDSA: use `app.addContentTypeParser('application/json', { parseAs: 'string' }, ...)` within the plugin scope — stores raw body on `(req as any).rawBody`, then parses JSON manually; does not affect other routes' parsers due to plugin encapsulation
- Fire-and-forget `void processor.processBatch(events)` works correctly in vitest: the mock call is registered synchronously even though the promise is discarded
- Register webhook routes at root (no prefix) in buildApp: `await app.register(webhookRoutes)` so path is `/webhooks/sendgrid` not `/emails/webhooks/sendgrid`
