## 2026-04-02 - US-008
- Implemented JWT auth Fastify plugin with static RS256 and JWKS modes
- Uses fast-jwt createVerifier with RS256 algorithm
- Static mode: verifies with PEM public key from IDENTITY_SERVICE_PUBLIC_KEY env
- JWKS mode: fetches keys from IDENTITY_SERVICE_JWKS_URL, matches by kid, caches for 5 minutes
- Both modes validate expiry, iss (if JWT_ISSUER set), aud (if JWT_AUDIENCE set)
- Invalid/expired token returns 401 { error: 'Unauthorized' }
- Plugin registered via fastify-plugin (fp) for proper encapsulation
- Files created: apps/platform/integration-hub/src/plugins/jwt-auth.ts
- **Learnings for future iterations:**
  - fast-jwt createVerifier with a string key returns a sync verifier; with an async KeyFetcher returns async verifier
  - JWKS keys need to be converted from JWK format to PEM using Node.js crypto createPublicKey
  - Plugin exports JwtAuthOptions interface for configuration by app.ts (US-024)
  - Webhook routes must NOT have this plugin applied — register them on a separate scope
