## Codebase Patterns
- Repository files follow `{table-name}.repo.ts` naming, export a `TABLE` constant, use `Knex` param, and cast rows with `as Interface`
- Use `onConflict(['col1', 'col2']).ignore()` for upsert-with-ignore pattern in Knex
- No `withSchema()` in repo queries â€” the search_path handles schema resolution at connection level
- Integration tests use `describe.skipIf(!HAS_DB)` guard pattern
- Migrations use `withSchema('crm_campaigns')` but repos do not
- `bulkInsert` omits sent_at (defaults to NULL); use `bulkInsertFull` when rows need explicit sent_at values (e.g., A/B recipient inserts)
- A/B split helper: `sendVariant()` private function in send-orchestrator.ts handles Email Service POST for any variant
- Export `processJob` from worker files to enable direct invocation in integration tests without BullMQ
- When a worker needs to enqueue jobs on another queue, import `Queue` from bullmq and instantiate with the target queue constant
