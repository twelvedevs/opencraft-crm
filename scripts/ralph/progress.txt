## Codebase Patterns
- Platform services use ESM (`"type": "module"`) with NodeNext module resolution — always use `.js` extensions in imports
- tsconfig follows automation service pattern: target ES2022, NodeNext, strict, outDir dist/, rootDir src/
- Fastify app is exported from index.ts; server only starts when NODE_ENV !== 'test' (allows testing without binding port)
- config.ts uses requireEnv() helper that throws on missing required vars; PORT has default 3006
- BullMQ and ioredis are separate dependencies (ioredis used for pub/sub, BullMQ for job queues)
- Services run on port 3006 (notification), matches existing port allocation pattern
