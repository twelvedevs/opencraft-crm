## Codebase Patterns
- spamscanner (npm) uses `scanner.scan(rfcEmailString)` not `scanner.getReport()` — build a minimal RFC 2822 string from subject/html/text; map `classification.probability * 10` to a 0–10 score
- spamscanner ships its own TypeScript types at `dist/types/index.d.ts`; no extra @types package needed
- Repositories follow TypeBox schema + Static type pattern: define `*Schema` with `Type.Object(...)`, export `type * = Static<typeof *Schema>`, then export the class
- Repository constructors take `private readonly db: Knex`; all queries use `this.db('table_name')`
- Nullable columns use `Type.Union([Type.String(), Type.Null()])` in TypeBox schemas
- `returning('*')` + cast `row as T` for insert operations
- Use `this.db.fn.now()` for timestamp fields in updates
- `app.decorate()` adds typed fields to FastifyInstance; corresponding module augmentation lives in `types.d.ts`
- Routes registered via `app.register(plugin, { prefix: '/emails' })` in `buildApp()`
- ioredis: use `import { Redis } from 'ioredis'` (named export) — default import `IORedis` cannot be used as a type
- BullMQ requires `maxRetriesPerRequest: null` on the IORedis connection
- When `buildApp` signature changes, update `app.test.ts` stubs (all 3 call sites)
- BullMQ worker tests: use top-level `vi.mock('bullmq', () => ({ Worker: vi.fn() }))`, capture processor + failed handler in `beforeEach` by setting `Worker.mockImplementation` before calling `createWorker()`
- Worker processor is tested by calling `capturedProcessor({ data: jobData })` directly — no real BullMQ queue needed
- Type the `publishSpy` as `any` when using `vi.spyOn` on an EventBus method to avoid complex `MockInstance` variance errors
