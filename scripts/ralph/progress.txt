## Codebase Patterns
- Tests use Vitest with `vi.fn()` for mocks; import from `vitest` directly (not jest)
- All imports use `.js` extension (ESM module resolution, even for `.ts` source files)
- `RulesRepository` lives in `src/repositories/rules.repository.ts` — extend it in place, do not create new repo files
- `Date.now()` can be spied on with `vi.spyOn(Date, 'now')` to advance TTL in tests
- TypeBox: use `Value.Check(schema, val)` for boolean validation; `Value.Errors(schema, val)` for error details (spread into array); import `Value` from `@sinclair/typebox/value`
- EventBridge envelope: only `detail` field matters — all other envelope fields (id, source, time, region) are ignored
