## Codebase Patterns
- Test files in `test/unit/` import from `../../src/services/<module>.js` (ESM `.js` extension required even for `.ts` sources)
- The project uses `"type": "module"` + NodeNext resolution — always use `.js` extensions in imports
- `vitest run` executes all test files under `test/`; no extra config needed
- Pure service modules go in `src/services/`; no registration in index.ts needed for pure functions
- DOT_PATH_RE must be checked before TEMPLATE_TOKEN_RE since dot-paths won't contain `{{}}` but template strings might match path-like patterns
- Use `Intl.DateTimeFormat` with `hourCycle: 'h23'` for timezone-aware time-of-day extraction; no third-party library needed
- For active-hours tests: inject a fixed `now` Date using UTC-aligned timestamps so wall-clock time == UTC clock time; avoids test flakiness
