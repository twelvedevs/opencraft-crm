## Codebase Patterns
- Email service uses `src/env.ts`; template service uses `src/config.ts` per PRD spec
- All services use TypeBox `Type.Object` + `Value.Check` for env validation; extract first error path for message
- `buildApp(db, ...)` factory pattern — app does NOT call listen; `index.ts` owns the listen call
- Guard `app.listen()` with `NODE_ENV !== 'test'` to allow test imports without binding a port
- `src/types.d.ts` augments `declare module 'fastify'` for typed decorators
- tsconfig: `target ESNext`, `module NodeNext`, `moduleResolution NodeNext`, `strict true`
- `lru-cache` and `jose` are in dependencies (not devDependencies) — they're runtime deps
