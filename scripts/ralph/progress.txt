## Codebase Patterns
- Always add `.gitignore` with `node_modules/` and `dist/` before staging service files â€” `git add apps/platform/<service>/` will otherwise pull in node_modules
- ESM throughout: `"type": "module"` in package.json, `module: NodeNext` + `moduleResolution: NodeNext` in tsconfig.json
- Fastify v5 with `@fastify/sensible`; use `logger: true` for pino â€” no pino-pretty dep needed
- `tsc --noEmit` is the typecheck command (via `npm run typecheck`)
- Knex migrations: use `withSchema('platform_automation')` on every `knex.schema` call; create schema in migration 001 with `knex.raw('CREATE SCHEMA IF NOT EXISTS platform_automation')`
- knexfile.ts lives in `migrations/` alongside migration files; set `directory: '.'` in migrations config
- tsconfig.json `include` only covers `src/**/*` â€” migration files are excluded from typecheck by design (avoids rootDir conflict)
