## Codebase Patterns
- Workspace packages (@ortho/logger, @ortho/auth-middleware, @ortho/event-bus) must be built (`npm run build`) before consuming services can typecheck — they resolve via `file:` references to `dist/`
- Use `node:24-slim` (not alpine) for media service Dockerfile — required for sharp prebuilt binaries
- Follow identity service patterns: `buildApp(pool)` factory, `createLogger('platform-media')`, cast logger `as unknown as FastifyBaseLogger`
- tsconfig uses `module: NodeNext` / `moduleResolution: NodeNext` (not ESNext/bundler) to match other services
- All imports use `.js` extension for ESM compatibility (e.g. `./env.js`)
- Repository functions are standalone exports (not class-based), with `knex: Knex` as first parameter and fully-qualified table names (`'platform_media.media_files'`)
- PostgreSQL bigint columns come back as strings via Knex — type them as `string | null`
