## Codebase Patterns
- Each service/package is standalone with its own node_modules (no root workspace manager)
- Use `file:../../../packages/@ortho/<name>` for local package references in package.json
- @ortho/event-bus must be built (`npm run build`) before dependent packages can typecheck
- TypeBox `Value.Check` + `Value.Errors` is cleaner than `Value.Parse` for env validation with custom error messages
- Fastify app builder pattern: `buildApp(db: Knex, eventBus: EventBus): Promise<FastifyInstance>` — decorate instance, register hooks, register routes
- Register `onReady` hook (not `onStart`) to call `eventBus.start()` after Fastify is ready
- Module augmentation in `src/types.d.ts` is needed to type `fastify.db` and `fastify.eventBus` decorators
- `tsconfig.json`: no workspace base extension — standalone per service (same as event-bus package)
- Fastify TypeBox response schemas: define ALL used status codes (including error codes like 404, 409) in the `response:` schema object — otherwise `reply.status(N)` causes TS type errors
- `@fastify/sensible` v5 supports Fastify 4.x only — use `^6.0.0` for Fastify 5.x
- For tests that import modules with top-level env parsing, set env vars in `vitest.config.ts` under `test.env` (not setupFiles) — they are applied before module evaluation
