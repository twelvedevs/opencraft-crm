## Codebase Patterns
- Integration tests: `describe.skipIf(!HAS_DB)` pattern with helpers.ts (getDb, runMigrations, cleanup, truncateTables); setup.ts sets env vars before module imports; nock for HTTP mocking; `processImportJob` for direct phase invocation
- No `@ortho/db` package exists; each service creates a local `db.ts` with `knexLib({ client: 'pg', connection: env.DATABASE_URL, searchPath: ['<schema>', 'public'] })`
- Service env vars use `src/env.ts` with `required()` / `optional()` helper pattern — throws on missing required vars at import time
- Fastify setup: `createLogger('crm-<name>')`, cast as `unknown as FastifyBaseLogger`, `@fastify/sensible` registered, auth in scoped plugin
- Dockerfile: multi-stage with `node:24-alpine` (builder + runner)
- Package naming: `@ortho/<service>-service`
- All services use file: references for monorepo packages (e.g., `"@ortho/auth-middleware": "file:../../../packages/@ortho/auth-middleware"`)
- Knex migrations: use `string()` not `varchar()` (Knex types lack `varchar`); use `specificType('col', 'type[]')` for Postgres arrays; CHECK constraints and indexes via `knex.raw()` with fully-qualified `schema.table` names
- BullMQ Worker: connection uses `{ url: env.REDIS_URL }`, worker-level `removeOnComplete` is `KeepJobs` type (`{ count: 0 }` = true), omit `removeOnFail` to keep failed jobs
- Worker exports `ImportJobData` interface and `startWorker` function — routes enqueue jobs with `{ import_id, phase }` payload
- Route pattern: factory function `routeName(opts)` returning async Fastify plugin; `requireRole` for role arrays, `requireLocation` for location scoping
- `@fastify/multipart` needs side-effect `import '@fastify/multipart'` for `req.file()` type augmentation
- `libphonenumber-js` rejects `555` area codes — use real US area codes (212, 310, etc.) in tests
