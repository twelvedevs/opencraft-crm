## Codebase Patterns
- @fastify/cors ^5.0.0 doesn't exist; use ^11.0.0 for Fastify 5 compatibility
- Mirror apps/platform/ai/ for new service scaffolding (package.json, tsconfig.json, Dockerfile)
- @ortho/auth-middleware stub must exist at packages/@ortho/auth-middleware/ for file: dependency to resolve
- All services use ESM ("type": "module") with NodeNext module resolution
- Knex CHECK constraints require `knex.raw()` — schema builder doesn't support them
- Use `.specificType('col', 'varchar[]')` for Postgres array columns in Knex
- `vitest run` exits code 1 with no test files — expected before test stories land
- fast-jwt createVerifier accepts JWK objects directly — no jwk-to-pem conversion needed
- Auth0 user IDs contain pipes (auth0|xxx) — always encodeURIComponent in URL paths
- Register authPlugin in app.ts with IDENTITY_JWKS_URL env var and allowedPaths for unauthenticated routes
- Integration tests must mock global.fetch to serve JWKS before building the app (authPlugin fetches JWKS during registration)
- Do NOT set a default DATABASE_URL in test helpers — let describe.skipIf detect its absence
- Repo functions use `DbClient = Pool | PoolClient` type alias for transaction support
- All SQL queries must qualify tables with `platform_identity.` schema prefix
- Service errors use `throw new Error(msg)` with `statusCode` and optional `details` properties for HTTP response mapping
- fast-jwt VerifierSync is a `declare function` — use `ReturnType<typeof createVerifier>` for type annotations
- Node.js `crypto.createPublicKey({ key: jwk, format: 'jwk' })` converts JWK→PEM without external packages
- Build @ortho/auth-middleware (`npx tsc` in packages/@ortho/auth-middleware/) before identity typecheck — needed for `req.user` type augmentation
- Import `'@ortho/auth-middleware'` (side-effect) in route files that access `req.user`
- Internal-only routes use X-Internal-Secret header guard preHandler instead of JWT auth
- For modules with module-level env reads (e.g. token.service), use dynamic `await import()` in tests after setting process.env
- Mock fast-jwt createVerifier in tests to avoid JWK→PEM conversion issues — it doesn't accept raw JWK with `algorithms` option
