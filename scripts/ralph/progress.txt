## Codebase Patterns
- Use `{ Redis } from 'ioredis'` (named import, NOT default import) — default import fails typecheck
- Follow email service (`apps/platform/email/`) patterns for env.ts, db.ts, app.ts, index.ts, types.d.ts, health.ts
- TypeBox schemas use `Type.Union([Type.String(), Type.Null()])` for nullable columns
- Repositories follow class pattern with Knex constructor injection, `returning('*')` on inserts
- Migrations use `.ts` extension with `export async function up/down(knex: Knex)` pattern
- Use `knex.raw('gen_random_uuid()')` for UUID PKs, `knex.fn.now()` for timestamps
- Cursor-based pagination: base64-encode `{ created_at, id }` JSON for keyset pagination
- Route plugins: register WITHOUT prefix in app.ts, use full paths (e.g. `/numbers`) inside the plugin — email service `domainRoutes` uses this pattern
- Unique constraint violations: catch with Knex error code `23505` via `isUniqueViolation()` helper
- Knexfile: use `export default { ... }` for ESM compatibility
- ioredis `scriptLoad` doesn't exist — use `redis.script('LOAD', luaScript) as string` for SCRIPT LOAD
- buildApp signature: `(db, eventBus, redis, twilioClient, statusCallbackUrl)` — all 5 params required
- vitest.config.ts: use `passWithNoTests: true` to avoid exit code 1 when no test files exist
- TwilioClient uses interface + stub pattern for DI — stub is co-located in twilio-client.ts
- Twilio webhook signature validation: HMAC-SHA1 with URL + sorted key-value params, base64 encoded, compared with timingSafeEqual
- Use `@fastify/formbody` for parsing `application/x-www-form-urlencoded` Twilio webhook bodies
- Webhook routes use preHandler for signature validation — keeps validation DRY across inbound + status routes
- Integration tests: use `checkInfra()` guard with `describe.skipIf(!infraAvailable)` to gracefully skip when Postgres/Redis aren't running
- Integration tests: use `EventBusImpl` + `MockDriver` from `@ortho/event-bus` — check `mockDriver.published` array for event assertions
- Integration tests: use `generateTwilioSignature()` helper to create valid HMAC-SHA1 signatures for webhook testing
- Integration tests: use `app.inject()` with `content-type: application/x-www-form-urlencoded` + `URLSearchParams.toString()` for webhook payloads
- Event publishing: wrap all `eventBus.publish()` calls in try/catch — log errors but don't fail the webhook response (non-critical)
